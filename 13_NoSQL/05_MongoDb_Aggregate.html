Gruppierungen ($group)

In SQL sind Aggregierungen auch (fast) immer mit entsprechenden Gruppierungen verbunden. Wollen wir die Anzahl der Schler pro Klasse zhlen, so kann dies mit folgenden SQL Statement formuliert werden:

SELECT KlasseId, COUNT(*), MIN(Gebdat) AS Juengster
FROM Schueler

Wollen wir nun in MongoDB die Collection Schueler nach diesen Informationen abfragen, so mtzen wir die Funktion aggregate nutzen. Aggregate fhrt die Pipeline aus, die in unserem Beispiel nur aus der Gruppierungsebene besteht.

Die Technik des Zhlens ist jedoch anders: Die Anzahl wird so ermittelt, dass fr jedes Dokument der Wert 1 aufsummiert wird. Die Aggregatsfunktionen knnen mit $sum und $min und dem Key, der fr die Berchnung verwendet wird, angegeben werden.

> use Stundenplan
> db.Schueler.aggregate([ 
    { $group: { _id: "$KlasseId", anzahl: { $sum: 1 }, juengster: {$min: "$Gebdat"} } }    
  ]);

Filtern mit $match und $group

Mit WHERE knnen wir in SQL vor der Gruppierung die Werte filtern:

SELECT KlasseId, COUNT(*) AS Anzahl
FROM Schueler
WHERE KlasseId LIKE '%HIF'

In diesem Beispiel bergeben wir einen regulren Ausdruck (*/HIFmatchdieMoglichkeit,inderPipelinedieseFilterungvorzunehmen.IndiesemBeispielubergebenwireinenregularenAusdruck(/HIF/), der bedeutet, dass die Klassen ID mit HIF enden muss.

> db.Schueler.aggregate([

Abbildung in LINQ

Zum Glck mssen wir als .NET Programmierer diese Methoden nicht hndisch aufrufen. Der MongoDB Treiber fr .NET erstellt aus den gewohnten LINQ Aggregationsmethoden wie GroupBy(), Count(), ... die entsprechende Pipeline.


_db.GetCollection<Schueler>(nameof(Schueler))
  .AsQueryable()
  .Where(s => s.KlasseId.EndsWith("HIF"))               // Erzeugt $match
  .GroupBy(s => s.KlasseId)                             // Erzeugt $group
  .Select(g => new { Id = g.Key, Count = g.Count() })   // Projektion
  .OrderBy(g => g.Count).ThenBy(g => g.Id)              // Erzeugt $sort
